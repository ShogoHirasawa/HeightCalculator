# 窓枠サイズ計測（屋内）開発仕様書

本書は、既存の「高さ計測アプリ（三角測量式）」（`height_measure_spec.md`）に**窓枠サイズ計測**機能を追加するための仕様書である。引越し見積もりで、地面〜2階ベランダ/窓までの高さ（屋外計測）に加え、**搬入経路となる窓枠の開口寸法（幅×高さ）を屋内から計測**できるようにする。

> 確定方針: 1フロー連続 / 四隅→幅×高さ＋対角線 / 壁の垂直面レイキャスト / 別計測＋写真保存・共有。
> 確定UI: モード切替は下部セグメント `[高さ][窓枠]`、四隅順序は 左上→右上→右下→左下、精度目標 ±0.05m、寸法ラベルは 幅×高さ＋対角を常時表示。

---

## 1. 目的

部屋の中から窓枠（開口部）の四隅をARで指定し、**幅・高さ・対角線**を計測する。家具・荷物が窓から搬入できるかの判定に用いる。高さ計測（屋外）と同一アプリ内で、連続したフローとして実施する。

---

## 2. スコープ

### 2.1 含むもの
- 窓枠の四隅を壁（垂直面）上に置いて、**幅・高さ・対角線**を算出（長方形前提）。
- 高さ計測（屋外）→窓枠計測（屋内）の連続フロー。結果は**別計測**として個別に保持・表示。
- 窓枠計測の結果（寸法を合成した画像）の撮影・保存・共有（§7.7 と同方式）。

### 2.2 含まないもの
- 台形・アーチ等の非長方形窓の厳密計測（長方形近似で代用）。
- 窓枠の奥行き（壁面より引っ込んだ深い枠）の厳密補正。
- LiDAR・メッシュ再構成・自動窓認識。
- 家具サイズDB照合による自動可否判定（寸法表示のみ。判定は人が行う）。

---

## 3. 全体フロー・モード（確定方針：1フロー連続）

アプリは2つの計測モードを持つ。`enum MeasureMode { case height, window }`。

- **モード切替**: 画面に計測モードの切替（例: 上部のセグメント `[高さ] [窓枠]`、または高さ確定後の「窓枠を計測へ」ボタン）。既定は `.height`。
- **推奨順序**: ①屋外で高さ計測 →（必要なら撮影・保存）→ ②屋内へ移動 → 窓枠モードへ切替 → 窓枠計測。順序は強制しない（どちらからでも可）。
- **結果の保持**: 高さの結果（数値）と窓枠の結果（数値）は**別々に保持**する。モードを切り替えても相互に消えない。各モードで撮影・保存・共有できる。
- **AR前提**: 屋外→屋内移動で世界トラッキングはドリフトするが、窓枠計測は屋内で改めて検出した壁面上の点同士の相対距離（数秒内）なので影響を実質受けない。**トラッキングはリセットしない**（高さ計測のワールド座標を保持し、モードを戻しても高さ表示が壊れないようにするため）。屋内では新たに垂直面が検出され、窓枠の角は `.vertical` レイキャストで取得する（屋外の床平面は対象外なので干渉しない）。
- ARセッション設定は既存どおり（`planeDetection = [.horizontal, .vertical]`、`worldAlignment = .gravity`）。窓枠では**垂直面（壁）**を主に用いる。

---

## 4. 計測アルゴリズム（窓枠・確定方針：壁レイキャスト）

高さ計測の三角測量とは別方式。窓枠の四隅を**壁（垂直面）にレイキャストして3D点として置き、点同士のユークリッド距離**で寸法を出す（Apple純正「計測」と同系統）。

### 4.1 四隅の捕捉
- 画面中央レティクルを窓枠の角に合わせ、計測ボタンで1点ずつ確定する。
- 各点は `arView.raycast(from:arView.center, allowing:.existingPlaneGeometry, alignment:.vertical)` を優先（実検出の壁面）。0件時は `.estimatedPlane` → `.existingPlaneInfinite` の順でフォールバック（高さ計測 §5.1 と同じ精度優先）。
- 確定した点のワールド座標を保持し、白いマーカー（リング＋点）を置く。
- **順序（固定）**: ① 左上 → ② 右上 → ③ 右下 → ④ 左下（時計回り）。
- 各辺・対角を結ぶ線分を白で描画する（確定済みの隣接点間）。

### 4.2 寸法の算出
4点 `TL, TR, BR, BL`（ワールド座標）から、`d(a,b)=|a−b|`（3Dユークリッド距離）として:

```
top    = d(TL, TR)        // 上辺
bottom = d(BL, BR)        // 下辺
left   = d(TL, BL)        // 左辺
right  = d(TR, BR)        // 右辺
diag1  = d(TL, BR)
diag2  = d(TR, BL)

幅   width    = (top + bottom) / 2     // 上下辺の平均
高さ height   = (left + right) / 2     // 左右辺の平均
対角 diagonal = (diag1 + diag2) / 2
```

- 結果は **幅 × 高さ（＋対角線）**。対角があるので「斜めに通すか」の判断にも使える。
- 表記: 1m 未満は cm、以上は m（小数2桁）。`HeightFormat.string` を共通利用。

### 4.3 純関数化（テスト用）
上記計算を、4点 `SIMD3<Double>` を受け取り `WindowSize(width,height,diagonal)` を返す純粋関数 `WindowCalculator.size(topLeft:topRight:bottomRight:bottomLeft:) -> WindowSize` として切り出す（ARKit/RealityKit 非依存、`simd` のみ）。これを§11のユニットテストで検証する。

### 4.4 内側の可視化（計測範囲のグレーアウト）
確定後（4点）、囲った四角形の**内側を控えめにグレーアウト**して計測範囲を一目で分かるようにする。
- 四隅を `ARView.project(_:)` で投影した四角形（時計回り）を**黒・不透明度0.22**で塗る（新しい色相は足さず、白い枠線・四隅マーカー・寸法ピルがその上でくっきり読めるようにする）。
- ライブ表示は SwiftUI の `Path.fill`（寸法ラベルより下に重ねる）。撮影画像も同じ四角形を Core Graphics で塗ってから寸法ピルを描く。
- 対角の数値ピルは中心に表示し続ける（塗りの上でも視認できる）。

## 5. 状態遷移（窓枠モード）

`enum WindowState { case detectingWall, placing(Int), done }`（`placing` は確定済み点数 0〜3）。

| 状態 | 条件 | 計測ボタン押下時 |
|---|---|---|
| `detectingWall` | 壁（垂直面）未検出 | ボタン無効 |
| `placing(n)`（n=0..3） | 壁検出済み・n点確定済み | レティクル位置を壁にレイキャストし n+1 点目を確定。4点目で `done` へ |
| `done` | 4点確定 | （計測ボタンは無効化。やり直し/クリア/撮影が可能） |

- やり直し: 直前に置いた点を1つ削除（`placing(n)` → `placing(n-1)`）。`done` からは4点目を削除して `placing(3)` に戻す。
- クリア: 全点・線・寸法を削除し `placing(0)` に戻す。

## 6. 画面・UI仕様（窓枠モード）

- **レティクル**: 既存の面追従レティクル（壁では立つ）。`.existingPlaneGeometry`（壁）にロック時は緑、未ヒットは2Dフォールバック。
- **上部バナー（状態別）**:
  - `detectingWall`: `窓のある壁を映してください`
  - `placing(0)`: `窓枠の左上の角に合わせてボタンを押してください`
  - `placing(1)`: `右上の角に合わせてください`
  - `placing(2)`: `右下の角に合わせてください`
  - `placing(3)`: `左下の角に合わせてください`
  - `done`: `幅 ○○ × 高さ ○○（対角 ○○）`
- **進捗表示**: 四隅インジケータ（①②③④を確定で塗り）。
- **計測ボタンラベル**: `① 左上 / ② 右上 / ③ 右下 / ④ 左下`（`placing(n)` に対応）。`detectingWall` は無効。
- **確定後（done）**: 窓枠の長方形アウトライン＋寸法ラベル（上辺に幅、左辺に高さ、中央付近に対角）を線上に常時表示。
- **やり直し / クリア / 撮影** ボタン（高さモードと共通の見た目）。
- ビジュアルは §7.5 の Apple HIG 準拠（マテリアル・SF Symbols・白基調の線/点/数値ピル）を踏襲。

## 7. エラー・例外処理（窓枠モード）

| 事象 | 検出条件 | 表示文言 |
|---|---|---|
| 壁未検出 | 確定時にレイキャスト結果0件 | `壁が検出できません。窓のある壁を映してください` |
| トラッキング不良 | `camera.trackingState` が `.notAvailable`/`.limited` | `動かさずに少し待ってください（トラッキング調整中）` |
| 退化した四角形 | 算出時、幅 or 高さ が極小（例 < 0.02m）または点が重複 | `角の位置がうまく取れません。離れて取り直してください` |

---

## 8. 撮影・保存・共有（窓枠・§7.7 準拠）

高さと同じ3ステップ（①計測 →②撮影〔ガイド付き〕→③保存/共有）。
- 撮影画像には**窓枠アウトライン（線・四隅マーカーはARエンティティ）＋寸法（幅×高さ、対角）の数値ピル**を合成する。
- 撮影ガイド文言（最小限）と状態チップ（例: `窓枠` が画角に入ると緑）。
- 高さ計測の写真とは別に保存・共有できる。

## 10. 既知の制約

- **深い窓枠**: 枠が壁面より奥に引っ込んでいると、壁面投影との差が誤差になる。
- **無地の壁**: 凹凸・テクスチャが乏しいと垂直面検出が弱い。窓枠の縁・壁紙の模様等があれば実用上可。
- **ガラス反射**: ガラス面は特徴点が出にくいが、計測対象は枠の角なので枠/壁のテクスチャがあれば可。
- 長方形近似のため、非長方形窓は近似値。

## 11. 受け入れ基準（窓枠）

1. **ユニットテスト**: `WindowCalculator` に既知4点入力で期待値を検証するテストを最低2ケース（許容誤差 ±0.01m）。
   - 例: 平面上の長方形 `TL=(0,1.5,0)`, `TR=(1.2,1.5,0)`, `BR=(1.2,0,0)`, `BL=(0,0,0)` → 幅 1.20m / 高さ 1.50m / 対角 √(1.2²+1.5²)=1.921m。
2. ビルドがエラーなく成功し、既存テストと併せて `swift test` が全緑。
3. 窓枠モードで四隅を順に置くと長方形アウトラインと寸法が表示され、やり直し/クリアが機能する。
4. 高さの結果と窓枠の結果が別々に保持され、各モードで撮影・保存・共有できる。
5. （実機）既知サイズの窓枠（実測値）を屋内から計測し、表示値が実測の **±0.05m** 程度に入る（精度目標・撮影条件で変動。受け入れはこの基準）。

> 実機・カメラ・ARKitの実動作精度は無人検証対象外（既存方針どおり、人が実機で確認）。検証ゲートはビルド成功＋ユニットテスト。

## 12. 実装メモ

- モード切替は下部セグメント `[高さ][窓枠]`（`OverlayView.modeSwitcher`）。窓枠へ切替時は窓枠状態をクリアしレティクルを作り直す（トラッキングはリセットしない）。
- 窓枠の四隅順序は固定（左上→右上→右下→左下、時計回り）。計測ボタンは `① 左上`〜`④ 左下`、壁ロック時のみ有効。
- 寸法ラベルは確定後、幅（上辺中点）/高さ（左辺中点）/対角（中心）に白カプセル＋黒文字で常時表示し、撮影画像にも同位置で合成する。
- 純関数 `WindowCalculator.size(...)` は `WindowSize?`（退化時 nil）。SPM/Xcode 両方にユニットテストを追加済み。

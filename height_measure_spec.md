# 高さ計測アプリ（三角測量式）開発仕様書

本書は、iOS向けAR高さ計測アプリの最小プロトタイプを実装するための仕様書である。実装担当（Claude Code）は本書の記述のみを根拠に開発する。本書で「未定」「任意」とは書かない。判断が必要な箇所はすべて確定値を記載している。

---

## 1. 目的

スマートフォンを地面に立った状態でかざし、2階のベランダや窓など「地面より高い対象」までの高さ（地面からの鉛直距離）を計測する。LiDARを使わず、ARKitのワールドトラッキング（VIO）とIMUによる重力基準姿勢のみで計測する。引越し見積もりにおける吊り上げ要否判定の検証が想定用途だが、アプリ自体は汎用の「地面からの高さ計測ツール」として独立して動作する。

---

## 2. スコープ

### 2.1 本プロトタイプに含むもの

- 三角測量式による「地面からの高さ」計測（計測方式は §5 で確定）。
- 1回の起動セッション中に複数回計測でき、結果を画面上のリストに蓄積表示する。
- 計測結果のクリア機能。

### 2.2 本プロトタイプに含まないもの（将来対応・実装しない）

- 計測結果の保存・永続化（アプリ終了で全結果を破棄する）。
- 写真撮影・画像保存・共有・エクスポート。
- VLM等によるベランダ/窓の自動認識。
- 2点間の汎用直線距離計測。
- LiDARの利用、メッシュ再構成。
- Android対応、iPad最適化。
- ログイン、アカウント、サーバ通信、解析SDK。

## 4. 座標系と前提（確定）

- ARKitワールド座標系（右手系、メートル単位、`worldAlignment = .gravity` によりY軸が鉛直上向き）を用いる。
- 「地面」は `planeDetection = [.horizontal]` で検出した水平面とする。計測の高さ基準は、後述の「底点」を地面にレイキャストして得たワールド座標のY値とする。別途グローバルな地面Y値は保持しない。
- 計測前提: 対象（ベランダの縁など）は、地面上に指定した「底点」の鉛直線上にあると近似する。対象が壁面より手前に張り出している場合（バルコニーの張り出し等）はその張り出し分だけ誤差となる。これは本プロトタイプの既知の制約として許容する。

## 5. 計測アルゴリズム（確定）

ユーザは「① 壁の根元（対象の真下の地面）」と「② 対象（高さを測りたい点）」の2点を、画面中央のレティクルに合わせて順に捕捉する。各捕捉は下部の「計測ボタン」押下で行う（画面タップ位置には依存しない）。

### 5.1 ステップ① 底点の捕捉

- 画面中央点 `arView.center` から水平面へレイキャストする。**精度優先**で次の順に試し、最初に当たった結果を採用する（`alignment:.horizontal`）。
  1. `.existingPlaneGeometry`: 実検出された床の**範囲内のみ**。十字が指す“その場の床”に正確に当たるため、屋内・近距離で対象の真下を正確に取れる。
  2. `.estimatedPlane`: 特徴点からの推定平面。床がまだ十分検出されていない箇所の保険。
  3. `.existingPlaneInfinite`: 検出済み水平面の**無限延長**。遠く・浅角（地平線近く）の屋外の根元など、上記が当たらない場合の最後の保険。
  - **理由**: 無限延長を最優先にすると、十字が床より少し上（柱などの垂直面）を指したときにレイがオーバーシュートして対象の真下ではない床に当たる、複数の床（段差）で誤った面に当たる、等の誤配置が起きる。実検出された床を最優先にすることで近接・屋内の精度を確保しつつ、無限延長を最後の保険に回して遠方の根元も拾えるようにする。
  - **トレードオフ**: 無限延長が使われるのは平地前提のケースのみ。傾斜・段差や極端な浅角・遠距離では誤差が増えるが、本プロトタイプの前提では許容する。
- 最初のヒット結果のワールド座標を底点 `B = (Bx, By, Bz)` とする。`By` が高さ基準（地面）となる。
- ヒット結果が0件のとき（無限平面・推定平面いずれも）: §8 のエラー処理「床未検出」を表示し、底点は確定しない。
- 確定した `B` の位置に、純正「計測」アプリ風の**白いポイント**（床に寝かせた白いリング＋中心の白い点、`UnlitMaterial(.white)`）をAnchorEntityで配置する。

### 5.2 ステップ② 対象の捕捉と高さの算出

対象捕捉ボタン押下時の最新フレーム `arView.session.currentFrame` から以下を取得する。

- カメラ変換行列 `T = currentFrame.camera.transform`
- カメラ位置 `C = (Cx, Cy, Cz)` = `T.columns.3` の xyz
- カメラ前方ベクトル `f = (fx, fy, fz)` = `normalize(-(T.columns.2 の xyz))`（カメラはローカル−Z方向を向くため符号反転する）

水平距離・仰角・高さを次式で算出する。

```
d      = sqrt((Cx - Bx)^2 + (Cz - Bz)^2)          // 底点までの水平距離
f_h    = sqrt(fx^2 + fz^2)                          // 前方ベクトルの水平成分
tanθ   = fy / f_h                                   // 仰角の正接
H      = (Cy - By) + d * tanθ                       // 地面からの高さ（m）
```

- `H` が計測結果（地面からの対象の高さ、メートル）である。
- 有効条件（いずれかを満たさない場合は §8 の該当エラーを表示し、結果を確定しない）:
  - `f_h >= 0.05`（真上付近を向きすぎていない）
  - `fy > 0`（対象がカメラより上方にある）
- 算出後、底点 `B` から対象高さまでの鉛直線（`B` から `(Bx, By + H, Bz)` まで）をAR空間に描画する（線分の `ModelEntity`、太さ0.005m、**白** `UnlitMaterial(.white)`）。底点の白いポイントは残す。

### 5.3 高さ計算の純関数化

上記 `H` の計算は、入力 `C, f, B` を受け取り `H` を返す純粋関数 `HeightCalculator.height(camera:forward:base:) -> Double?`（無効条件のとき `nil`）として切り出し、ユニットテスト可能にする。

## 6. 状態遷移（確定）

アプリは単一画面で、以下の状態を持つ。状態は `ObservableObject` の `enum MeasureState` で管理する。

| 状態 | 条件 | 計測ボタン押下時の動作 |
|---|---|---|
| `.initializing` | ARセッション開始直後〜水平面未検出 | ボタン無効。 |
| `.waitingBase` | 水平面を1つ以上検出済み | §5.1 を実行し、成功で `.waitingTarget` へ。 |
| `.waitingTarget` | 底点確定済み | §5.2 を実行し、成功で結果をリストに追加して `.waitingBase` に戻る。 |

- `.initializing` → `.waitingBase` は、`session(_:didAdd:)` 等で水平面 `ARPlaneAnchor` を初めて検出した時点で遷移する。
- 1件の計測完了後は `.waitingBase` に戻り、次の計測を開始できる（底点マーカー・鉛直線は画面に残す）。

## 7. 画面・UI仕様（確定）

縦1画面。背景は全面ARカメラ映像（`ARView`）。その上にSwiftUIのオーバーレイを重ねる。

### 7.1 レイアウト要素

1. レティクル: 画面中央に十字またはリング。色 `#FFFFFF`、サイズ 32pt、不透明度0.9。常時表示。
2. 上部インストラクションバナー: 画面上端から下に配置。半透明の暗色背景（黒・不透明度0.5）に白文字。状態別テキストは §7.2。
3. 結果リスト: 画面下部、ボタンの上。各行 `計測 {n}: {H} m`（`{n}` は1始まりの連番、`{H}` は小数2桁）。最新の計測を最上段に表示する。表示は最大5件まで可視とし、超過分はスクロール可能にする。
4. 計測ボタン: 画面下端中央の大型ボタン（高さ56pt以上、横幅は画面幅の60%）。ラベルは状態別（§7.2）。`.initializing` では無効（不透明度0.4・タップ不可）。
5. クリアボタン: 画面下端右側の小ボタン。ラベル `クリア`。押下で全計測結果（リスト・AR上の底点マーカーと鉛直線エンティティ）を削除し、`.waitingBase` に戻す。
6. やり直しボタン: 画面下端左側の小ボタン。ラベル `やり直し`。`.waitingTarget` のときのみ有効。押下で直前に置いた底点マーカーを削除し、その計測を破棄して `.waitingBase` に戻す。`.waitingTarget` 以外では無効（不透明度0.4）。

### 7.2 状態別テキスト（確定の表示文字列）

| 状態 | バナー文言 | 計測ボタンラベル |
|---|---|---|
| `.initializing` | `床（地面）を映してください` | `準備中…` |
| `.waitingBase` | `壁の根元（対象の真下の地面）に照準を合わせて「計測」を押してください` | `① 壁の根元を指定` |
| `.waitingTarget` | `対象（ベランダの縁など）に照準を合わせて「計測」を押してください` | `② 高さを計測` |

### 7.3 結果確定時のフィードバック

- 計測成功時、結果行をリスト最上段に追加し、0.3秒間その行の背景を `#E1F5EE` でハイライトする。

### 7.4 床ロックガイド（誤選択防止）

①（壁の根元）で、十字が壁など床以外を指したまま計測してしまう誤操作を防ぐためのガイド。`.waitingBase` のときのみ動作する。

- 毎フレーム、画面中央から水平面へレイキャストし（§5.1 の精度優先順）、状態 `reticleState` を判定する。
  - `locked`: `.existingPlaneGeometry`（実検出の床）にヒット = 正確。
  - `approximate`: 推定平面・無限延長にのみヒット = おおよそ（遠方/未検出）。
  - `off`: いずれもヒットせず（壁・空中など）。
- **面に沿うレティクル（純正「計測」アプリ風）**: AR空間に白いリング（トーラス、外径約0.045m）＋中心の白い点を生成し、ヒットした面に `worldTransform` ごと適用して**面に沿って表示**する。レイキャストは `.any` アラインメントで行い、**床（水平）では寝かせ、壁（垂直）では立てて**表示する（見る角度に応じてレティクルが面の傾きに追従）。`UnlitMaterial` の白で表裏問わず視認できる。床にロックした瞬間（`off` → `locked`）に軽いハプティクスを出す。
- **計測可否は床（水平）のみ**: レティクルは壁にも沿うが、`reticleState` を `locked`/`approximate` にするのは `targetAlignment == .horizontal`（床）のときだけ。壁（垂直）は `off` とし、計測ボタンは無効のまま（壁を底点にしない）。
- **どの面にも当たらない時**: ARレティクルは非表示にし、画面中央に控えめな2D白リング（径約32pt・不透明度低）をフォールバック表示する。`.waitingTarget`・`.initializing` も2Dレティクルを表示する。
- **計測ボタンのガード**: `.waitingBase` では `reticleState != .off`（床を捉えている）ときのみ有効。`off` のときは無効化して壁などの誤選択を防ぐ。`.waitingTarget` は対象（上方）を狙うため常に有効、`.initializing` は無効。
- 旧版にあった「色付きフロアプレビューの平面マーカー」「床ロックチップ」は廃止し、床に沿うレティクルに一本化した。

### 7.5 ビジュアル（Apple HIG 準拠）

§7.1〜7.4 の機能・文言・寸法（ボタン高さ56pt以上等）・結果行フォーマット `計測 {n}: {H} m` は維持しつつ、見た目は Apple Human Interface Guidelines に準拠する。

- 背景は半透明マテリアル（`.ultraThinMaterial`）＋角丸でブラー表現。フラットな黒不透明背景は用いない。
- アイコンは SF Symbols（例: バナー `viewfinder` / `exclamationmark.triangle.fill`、計測ボタン `scope`・`arrow.up`、やり直し `arrow.uturn.backward`、クリア `trash`、結果行 `ruler`）。
- 計測ボタンはアクセント色 `#1D9E75` のカプセル（無効時はマテリアル＋不透明度低下）。やり直し・クリアは円形アイコンボタン＋下部に小ラベル。
- 数値は等幅数字（monospacedDigit）。エラー時のバナーは赤 `#A32D2D`。

### 7.6 ライブガイド（対象捕捉中の高さプレビュー）

底点 `B` 確定後（`.waitingTarget`）、純正「計測」アプリのように、確定前から現在の高さを可視化する。

- 毎フレーム、現在のカメラ位置 `C`・前方ベクトル `f` と `B` から §5.2 と同じ式で**暫定の高さ** `H` を算出する（`HeightCalculator.height`、無効角度のとき非表示）。
- **終点を鉛直線上に拘束する**: 終点を `T = (Bx, By + H, Bz)`（`B` の真上・高さ `H`）とし、`B` と `T` を `ARView.project(_:)` で画面に投影する。線・終点マーカーはレティクル（画面中央＝向いている方向）ではなく `T` に対して描く。これにより、ユーザーが左右にパンしても線は常に鉛直（数式上 `H` は方位に依存しない）で、**上下（ピッチ）で高さを合わせるだけ**でよい。
- **鉛直線は常時表示**（純正「計測」アプリ風の**白い実線**）: `.waitingTarget` に入った時点から線を表示する。角度が有効なときは `B` から終点 `T` まで、無効（カメラ未上昇）なときは `B` から真上のリファレンス上端（高さ `max(H+0.3, 0.6) m`）まで引く。これによりカメラを上に向ける前から鉛直ガイドが出る。`.waitingTarget` 中は画面中央の2Dレティクルは隠す。
- 角度が有効（対象がカメラより上）になったときだけ、鉛直線上の高さ `H` の点 `T` に**終点マーカー**（白いリング）と、`B`〜`T` 中点に**数値ピル**（純正Measure風の**白いカプセル＋濃い文字**）を重ねる。表記は 1m 未満が `cm`（例 `45 cm`）、1m 以上が `m`（小数2桁、例 `1.23 m`）。
- これらは確定（②押下）前のプレビューであり、計測結果には影響しない。②成功・やり直し・クリア・状態遷移で消える。

## 8. エラー・例外処理（確定）

エラーは上部バナーに2.0秒間、赤背景（`#A32D2D`・白文字）で表示し、その後 §7.2 の通常文言に戻す。状態は遷移させない（同じ状態で再試行可能）。

| 事象 | 検出条件 | 表示文言 |
|---|---|---|
| 床未検出 | §5.1 のレイキャスト結果が0件 | `床が検出できません。地面を映してから再度お試しください` |
| 対象が上を向きすぎ | §5.2 で `f_h < 0.05` | `角度が急すぎます。少し下げて対象に合わせてください` |
| 対象が下方 | §5.2 で `fy <= 0` | `対象はスマホより上に合わせてください` |
| トラッキング不良 | `currentFrame.camera.trackingState` が `.notAvailable` または `.limited` | `動かさずに少し待ってください（トラッキング調整中）` |

- トラッキング不良時は計測ボタンを押しても計測を行わず、上記文言を表示する。

## 10. 受け入れ基準（確定）

実装は以下をすべて満たすこと。

1. ユニットテスト: `HeightCalculator` に対し、既知入力で期待値を検証するテストを最低3ケース用意し、すべて成功する。
   - 例: `C=(0,1.5,0)`, `B=(0,0,3)`（水平距離3m、カメラ高1.5m）、`f` が仰角45°（`f=(0, √0.5, -√0.5)` を正規化したもの）のとき `H ≈ 4.5 m`（=1.5 + 3×tan45°）になること（許容誤差 ±0.01m）。
2. 実機での基準計測: 高さが既知の対象（例: 標準的なドア上端を実測した値、または壁に貼った基準点）を、地面から約3mの位置で計測し、表示値が実測値の ±0.20m 以内に入る。これは精度目標であり、撮影条件・端末により変動しうるが、受け入れ判定はこの基準で行う。
3. `.initializing` 中は計測ボタンが無効で、水平面検出後に `.waitingBase` へ自動遷移し有効化される。
4. 1セッションで連続3回以上計測でき、結果がリストに連番で蓄積され、最新が最上段に表示される。
5. クリアボタンで全結果（リスト・AR上の球マーカーと鉛直線）が消え、連番が1にリセットされる。
6. やり直しボタンが `.waitingTarget` でのみ有効で、押下により直前の底点が破棄され `.waitingBase` に戻る。
7. §8 の各エラー文言が、対応する条件で表示され、状態が遷移しないこと。
8. 縦向き固定で起動し、横向きでレイアウトが崩れない（横向き自体が無効）。
9. アプリを再起動すると結果が空になる（永続化していないこと）。

## 11. 補足（実装外・運用）

- 署名・配布（Apple Developer Program加入、Bundle Identifier設定、TestFlight配信）は本コード仕様の対象外。Bundle Identifierは仮値 `com.example.heightmeasure` をプレースホルダとして用い、実値はビルド設定で差し替える。
- 計測の数式・座標系（§4, §5）は本プロトタイプの確定仕様であり、精度改善（張り出し補正、複数フレーム平均化など）は将来課題として実装しない。

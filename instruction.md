height_measure_spec.md を読み、そこに定義された iOS アプリを一から実装してください。

# 進め方
1. CLAUDE.md の Git ワークフローに従い、main に直接コミットせず feat/ ブランチ（例: feat/height-measure-mvp）を作成して作業する。
2. Xcode プロジェクトは `HeightMeasure` という名前で新規作成する（スキーム名も HeightMeasure になる）。
3. 仕様書 §9 のファイル構成・クラス責務のとおりに実装する。
4. 変更のたびに CLAUDE.md「検証ゲート」記載のビルド・テストコマンドを実行する。
5. 次のすべてが緑になるまで、実装→ビルド→テスト→修正を止めずに反復する:
   - ビルドがエラーなく成功する
   - ユニットテストがすべて成功する（§10-1 の3ケースを含む）
   - 仕様書 §10「受け入れ基準」の各項目を満たす
6. すべて緑になったら、リポジトリ直下の .done に ALL_GREEN の一行のみを書き込み、変更内容を簡潔に要約する。

# 重要（無人実行のため）
- これは無人で実行されます。CLAUDE.md §1 の「不明点は止めて質問する」は本タスクでは適用しません。質問で停止せず、仕様書の前提・既定値（§3・§4・末尾の補足）に従って合理的に判断し、判断した内容はコミットメッセージに残してください。
- 実機・カメラ・ARKit の実動作（高さ計測の精度）の検証は不要です。朝に人間が実機で確認します。検証はビルド成功とユニットテストに限定してください。
- HeightCalculator は RealityKit / ARKit に依存させず、ユニットテストターゲットはホストアプリ無し（logic tests）で構成し、ARView を起動せずにテストできるようにしてください。
